# `map`

> Interactive map (MapLibre, no token) of one or more layers, each bound by a UDF (shorthand) or an explicit DuckDB `sql` query returning geometry as a GeoJSON-string column. Optionally emits the viewport bounds to a param. A subset of the application's `map`.

## Why
`map` paints an interactive geographic map (MapLibre basemap + deck.gl data overlay) of one or more layers — each layer bound by a UDF (shorthand) or an explicit DuckDB `sql` query that returns per-row geometry. An author reaches for it to visualize spatial output (points, lines, polygons) on a slippy map, optionally wiring the viewport bounds back into the param store to drive a downstream bbox-filtered query. Its ROLE is **source/display** (a multi-layer data-bound visualization), with an optional **input** facet (`sendParam` emits viewport bounds). App-parity: a strict SUBSET of the Fused application's `map` — same prop names/semantics (`label`, `param`, `sendParam`, `centerLng`/`centerLat`, `zoom`, `layers`, `mapStyle`, `style`), fewer props, MapLibre-with-no-token instead of Mapbox, and a narrowed `param` value type (see Expectation).

## Expectation
- Renders a map container with an optional label above it (when `label` set) and a map frame (8px corner radius, full width; the parsed `style` declaration is merged over the frame defaults). A MapLibre `Map` is created on mount with the basemap resolved from `mapStyle`, centered on `[centerLng, centerLat]` at `zoom`, plus a top-right `NavigationControl` (no compass).
- **Default height (no per-dashboard CSS hacks):** the frame carries a default height **FLOOR** of **360px** (via `.ofw-map__frame { min-height: 360px }` in `widget.css`); the inline frame style uses `height: 100%` (no fixed pixel height). The floor is a `min-height` on purpose — it survives the classic collapse where an author/canvas `height: 100%` resolves to **0** against an auto-height ancestor (which rendered the map blank), so the map paints one-shot at ≥360px without any wrapper `height` / `flex:1` hacks. **Author overrides still win:** `style` is parsed and spread **after** the base frame style (inline beats the CSS class), so an explicit `style="height:600px"`/`min-height` takes precedence, and a PARENT that pins an explicit height grows the map past the floor via the frame's `height:100%` (no canvas-sizing regression). (The sibling `fused-map` uses a taller 420px floor for its extra chrome.)
- DATA-BINDING is **per-layer**, not a single flat top-level `sql` prop. Each `layers[]` entry binds by EITHER `udf` (shorthand) OR an explicit `sql` (DuckDB, may carry `{{ref}}`/`$param`) — exactly one of the two. The planner normalizes each entry to an object and stamps `_queryId` + `_sql` onto it: a `udf` layer's `_sql` is the synthesized `SELECT * FROM {{udf}}`; a `sql` layer's `_sql` is the authored query. A non-rendering child per visible layer runs one `useDuckDbSqlQuery` against that layer's stamped `_sql`/`_queryId` (enabled only when a query id is present) and reports the resolved rows up. Use an explicit `sql` layer when the map must re-resolve as a `$param` changes (e.g. a slider/dropdown filtering the map).
- Rows are converted to GeoJSON features: each row's `vizConfig.geometryColumn` (default `"geometry"`) is read; a string value is `JSON.parse`d, a `Feature` is unwrapped to its `.geometry`, otherwise the value is treated as a raw geometry. If there is no usable geometry and both `vizConfig.latColumn` + `lngColumn` are set, a `Point` is built from those finite numbers. Rows with no valid geometry are skipped; all row keys become the feature's `properties`.
- One `MapboxOverlay` (deck.gl, `interleaved: true`) owns all layers; its layer set is rebuilt whenever any layer's data, `ready`, or `layers` change (deck reconciles by a per-layer id derived from `_queryId`, falling back to the layer's `udf` name). Each layer is a `GeoJsonLayer` (`pickable`, `pointType: "circle"`, pixel radius/line units). Layers with `visible === false` are filtered out of both the overlay AND the data-binding set.
- Styling from `vizConfig` (documented subset of the app's): static — `fillColor`, `lineColor` (default `lineColor ?? fillColor ?? "#E8FF59"`), `lineWidth` (1), `pointRadius` (5), `opacity` (0.8), `stroked` (true unless `false`), `filled` (true unless `false`), `geometryColumn`, `latColumn`/`lngColumn`. Data-driven — `radiusColumn` (+ `radiusRange` `[3,28]`, `radiusDomain`) scales point size by sqrt; `colorColumn` (+ `colorRange`, `colorDomain`) interpolates fill color across stops (requires ≥2 stops). Color parsing accepts `#rgb`/`#rrggbb`/`rgb()`/`rgba()`/an `[r,g,b,a]` array/a few named colors, defaulting to lime `[232,255,89]`.
- Hover tooltip: shows up to the first 6 row properties excluding `geometry` and any `_`-prefixed key, as a small dark monospace card; no tooltip when there are no displayable props.
- INPUT facet: when `sendParam` is true, a `moveend` listener emits the current viewport bounds to `param` via `useFusedParamWithForm<string>` as a `"west,south,east,north"` string (each coord `toFixed(6)`). `broadcastDefaultValue: false`, `debounceMs: 0`, `defaultValue: ""` — so nothing is broadcast until the map first moves. This is a **scalar string** value (SQL-safe; usable as `$param`).
- States: an in-frame `"Loading map…"` overlay while the dynamic `maplibre-gl` + `@deck.gl/*` imports / map `load` are pending; on import/init failure an in-frame `"Map failed: <msg>"` overlay (error is contained to the card, never blanks the widget). A `ResizeObserver` calls `map.resize()` on container resize; the map is `.remove()`d on unmount.
- Init is **once** — `mapStyle`/`centerLng`/`centerLat`/`zoom` are first-paint only and not reactive after mount.
- Deliberate behavioural subset vs the app: data binds **per-layer** (via `layers[].udf` or `layers[].sql`) rather than the app's flat form; `param`/`sendParam` emit a SQL-safe `"west,south,east,north"` **string** instead of the app's array form (same key, narrowed value); MapLibre with no token instead of Mapbox; `vizConfig` is a documented subset of the app's viz config.
- WHERE it renders: **native React host only** (the `widget-host/` viewer / parley, or an external control-plane consumer such as Flow, `fusedio/flow`). The self-contained deployed-serve bundle (`widget.html`) aliases this component to a placeholder that renders a "Map preview — maps render in a native widget host" notice instead of a map (the bundle ships no WebGL map libs, external tiles, or token channel).

## Exposed params

| prop | type | default | description |
|---|---|---|---|
| `label` | `string` | — | Label shown above the map. |
| `param` | `string` | — | Canvas param to receive the viewport bounds as a `"west,south,east,north"` string (SQL-safe); emitted when `sendParam` is true. |
| `sendParam` | `boolean` | — (renderer falls back to `false`) | Emit the viewport bounds to `param` on map move. |
| `centerLng` | `number` | — (renderer falls back to `-74.0`) | Initial center longitude. |
| `centerLat` | `number` | — (renderer falls back to `40.7`) | Initial center latitude. |
| `zoom` | `number` | — (renderer falls back to `12`) | Initial zoom level. |
| `layers` | `array<string \| {udf?: string, sql?: string, visible?: boolean, vizConfig?: record<string,unknown>}>` | — (required) | Map layers; each is a UDF name (string), or an object binding by `udf` OR `sql` (exactly one). `udf` is shorthand for `SELECT * FROM {{udf}}`; `sql` is a DuckDB query (may carry `{{ref}}`/`$param`) — use `sql` for a param-driven layer that re-resolves on slider/dropdown change. The query returns the layer's geometry as a GeoJSON-string column. `vizConfig` is a subset of the app's viz config (`fillColor`, `lineColor`, `lineWidth`, `pointRadius`, `opacity`, `geometryColumn` default `"geometry"`, …). |
| `mapStyle` | `enum("light", "dark", "satellite", "blank")` | — (renderer falls back to `"dark"`) | No-token basemap style. |
| `style` | `string` | — | Inline CSS declaration string, parsed and merged **over** the component's default frame styles (so an explicit `height`/`min-height` wins over the 360px default floor). |
| `layers[]._queryId` | `string` | — | (internal; resolver-stamped, not author-set) Per-layer query id stamped by the planner onto each normalized layer object. |

**Data-bound:** yes — per-layer via `layers[].udf` or `layers[].sql`; the planner stamps `_sql`/`_queryId` on each layer (`_sql` = the explicit `sql`, or the synthesized `SELECT * FROM {{udf}}`). The per-layer data binding reads all returned columns (geometry resolved from `vizConfig.geometryColumn`, default `geometry`; row keys become feature `properties`). There is no single top-level flat `sql` prop or top-level `_queryId`.

**Writes param:** no (`writesParam: false`). The component is declared non-input, but it DOES write the viewport bounds string to `props.param` at runtime via `useFusedParamWithForm` when `sendParam` is true (a scalar `"west,south,east,north"` string).

## Notes
- The map renderer dynamically loads MapLibre + deck.gl (`@deck.gl/mapbox`, `@deck.gl/layers`), runs one `useDuckDbSqlQuery` per visible layer, converts the resolved rows to GeoJSON features, parses colors, and draws every layer as a single `MapboxOverlay`. The component module itself stays thin and node-importable so the schema generator can load it without the heavy MapLibre/deck.gl deps.
- Deployed-bundle caveat: a single placeholder is the bundle stand-in for ALL map widgets (`map`, `map-bounds`, `fused-map`); at render the bundle reads only `.component`, so the placeholder's empty `props`/`writes: false` are irrelevant there — the generator loads the REAL module for the catalog.
- `param` is declared `writesParam: false` (so the generator's input lint does not flag the `param`+default combination), yet the renderer broadcasts on `sendParam`; consumers should still treat `param` as a scalar string source.
